## QA-1: 프라이버시 모드 토글 후 잠금화면 노출 확인

### 테스트 시나리오 1.1: Level 0 (제목 표시)

1. **설정 탭** → **프라이버시** 진입
2. **"제목 표시"** 선택 (체크마크 확인)
3. 앱을 백그라운드로 보내기
4. 잠금화면에서 위젯 확인

| 항목 | 예상 결과 | 실제 결과 | 통과 |
|------|----------|----------|------|
| 설정 저장 | 체크마크가 "제목 표시"에 표시됨 | | ☐ |
| 위젯 제목 | 태스크 제목이 원문 그대로 표시됨 (예: "회의 준비") | | ☐ |
| 메인 화면 표시 | 설정 화면에 "제목 표시"로 표시됨 | | ☐ |

## QA-2: 대표 프로젝트 변경 후 위젯 반영 확인

### 테스트 시나리오 2.1: 대표 프로젝트 설정

1. **설정 탭** → **대표 프로젝트** 진입
2. 프로젝트 하나 선택 (예: "업무")
3. 체크마크 확인
4. 뒤로 가기 → 메인 설정 화면 확인
5. 잠금화면 위젯 확인

| 항목 | 예상 결과 | 실제 결과 | 통과 |
|------|----------|----------|------|
| 선택 표시 | 선택한 프로젝트에 체크마크 표시 | | ☐ |
| 메인 화면 | "대표 프로젝트" 옆에 프로젝트명 표시 | | ☐ |
| 위젯 우선순위 | 해당 프로젝트의 태스크가 우선 표시됨 | | ☐ |

**메모**:
```
[여기에 특이사항 기록]
```

## QA-3: 위젯/단축어 가이드 화면 동작 확인

### 테스트 시나리오 3.1: 위젯 설정 방법 가이드

1. **설정 탭** → **도움말** 섹션
2. **"위젯 설정 방법"** 탭

| 항목 | 예상 결과 | 실제 결과 | 통과 |
|------|----------|----------|------|
| 화면 전환 | WidgetGuideView로 정상 이동 | | ☐ |
| 제목 표시 | "잠금화면 위젯 추가 방법" 헤드라인 표시 | | ☐ |
| 4단계 가이드 | 1~4단계 모두 표시됨 | | ☐ |
| 단계 번호 | 파란 원형 배경의 숫자 표시 | | ☐ |
| 면책 문구 | "위젯은 iOS 정책에 따라..." 문구 표시 | | ☐ |
| 스크롤 | 내용이 길 경우 스크롤 가능 | | ☐ |

**4단계 내용 확인**:
- [ ] 1: "잠금화면을 길게 누르세요"
- [ ] 2: "'사용자화'를 탭하세요"
- [ ] 3: "위젯 영역을 탭하세요"
- [ ] 4: "LivePlan을 찾아 추가하세요"

**메모**:
```
[여기에 특이사항 기록]
```

## QA-5: 엣지 케이스

### 테스트 시나리오 5.1: 프로젝트 없음 상태

1. 모든 프로젝트 삭제 또는 보관 처리
2. **설정 탭** → **대표 프로젝트** 진입

| 항목 | 예상 결과 | 실제 결과 | 통과 |
|------|----------|----------|------|
| 목록 표시 | "없음" 옵션만 표시됨 | | ☐ |
| 크래시 없음 | 앱이 정상 동작 | | ☐ |

---

### 테스트 시나리오 5.2: 대표 프로젝트 보관 시

1. 특정 프로젝트를 대표 프로젝트로 설정
2. 해당 프로젝트를 보관(archived) 처리
3. **설정 탭** 확인

| 항목 | 예상 결과 | 실제 결과 | 통과 |
|------|----------|----------|------|
| 폴백 동작 | "없음"으로 표시되거나 todayOverview 폴백 | | ☐ |
| 위젯 정상 | 위젯이 크래시 없이 표시됨 | | ☐ |
