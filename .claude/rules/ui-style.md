목적

LivePlan의 UI를 "가볍고 일관되게" 유지하기 위한 최소 스타일 규칙을 고정한다.

UI가 늘어나도 "기능이 무거워 보이지 않게" 하고, 화면별 구현 품질 편차를 줄인다.

적용 범위

SwiftUI 기반 앱 화면 전반

설정/온보딩/빈 상태/오류 상태 포함

상위 규칙 우선순위

product-decisions.md(기능/프라이버시 확정값)

ui-style.md(본 문서)

strings-localization.md(문구/키/길이)

error-and-messaging.md(에러 메시지 표준)

핵심 원칙(강행)

외부 UI 프레임워크 금지(Phase 1)

"디자인 시스템/테마 프레임워크" 도입 금지. 기본 SwiftUI 구성요소로 해결한다.

화면 구조는 "3~5개 핵심 컴포넌트"

한 화면에 정보 밀도를 과도하게 올리지 않는다.

"한 화면 = 한 목표"를 유지한다(프로젝트 목록, 프로젝트 상세, 설정 등).

빈 상태/오류/로딩은 반드시 정의

UX 품질을 좌우하는 상태이므로, 각 화면에 1개 이상의 empty state 문구와 CTA를 제공한다.

Dynamic Type/접근성 기본 준수

텍스트 크기 변화에서 레이아웃이 깨지지 않도록 과도한 고정 높이/너비 사용 금지.

버튼/토글/입력 필드는 접근성 레이블을 고려한다.

비목표

커스텀 폰트/복잡한 애니메이션/화려한 테마는 Phase 2+에서만

표준 컴포넌트 가이드(권장)
A. 리스트/항목

프로젝트 목록: List 또는 ScrollView+LazyVStack 중 하나로 통일

태스크 항목: "제목 + 상태(아이콘) + (선택) dueDate" 1~2줄로 제한

체크/완료 액션은 항상 같은 위치(우측 또는 좌측 고정)

B. 입력(프로젝트/태스크 생성)

필수 필드는 최소(프로젝트: title, startDate / 태스크: title, type)

유효성 실패 시 error-and-messaging.md 표준 메시지 사용

C. 설정 화면

섹션 3개 이하

프라이버시

대표 프로젝트

도움말(위젯/단축어 안내 링크)

D. 색/아이콘

의미 전달을 색상 하나에 의존하지 않는다(텍스트/아이콘 보조).

아이콘은 SF Symbols 우선.

E. 문구/표현

문구는 strings-localization.md 키를 통해 관리(하드코딩 금지 권장).

잠금화면 관련 문구는 lockscreen-copy-lab 결과를 우선 사용.

수동 QA(필수)

동적 글자 크기 2단계 이상에서 주요 화면(목록/상세/설정) 레이아웃 확인

빈 상태/오류 상태에서 CTA가 명확한지 확인

끝.
