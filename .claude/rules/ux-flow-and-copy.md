목적

LivePlan의 **핵심 사용자 흐름(온보딩/설정/매일 사용)**과 잠금화면 문구 기준을 단일 규칙으로 고정한다.

기능이 늘어나도 "사용자가 실제로 설정하고 매일 쓰는 흐름"이 흔들리지 않도록 한다.

적용 범위

앱 UI(프로젝트/태스크/설정)

잠금화면 표면(위젯/Live Activity/Controls)

단축어(Shortcuts) 및 인텐트 메시지(Refresh/Complete/QuickAdd)

온보딩/가이드/설명 텍스트 전반

상위 규칙 우선순위

.claude/rules/product-decisions.md (확정값)

.claude/rules/lockscreen.md (표면/우선순위/프라이버시)

.claude/rules/intents.md (인텐트 계약/멱등성/폴백)

.claude/rules/strings-localization.md (문구/현지화/길이 예산)

본 문서(ux-flow-and-copy)

핵심 원칙(강행)

2분 내 가치 노출

사용자가 앱을 처음 실행한 뒤 2분 내 "잠금화면에서 미완료/요약이 보이는 상태"를 만들 수 있어야 한다.

이 목표를 깨는 온보딩/설정 흐름은 금지한다.

설정은 "선택 기능"과 "필수 기능"을 분리한다

필수: 프로젝트/태스크 생성 + 잠금화면 위젯 추가 + 프라이버시 기본값(마스킹) 이해

선택: Live Activity 활성화, 단축어(8시간 갱신) 자동화 설정, Controls(iOS 18+) 배치

잠금화면은 "한눈에(요약)"

위젯/Live Activity는 상세 탐색을 제공하지 않는다.

상세는 탭 → 앱으로 이동이 기본이다.

문구는 짧고 일관되며 프라이버시 안전

프라이버시 기본값은 Level 1(마스킹)이다(product-decisions.md).

Level 1/2에서 원문 제목 노출 금지.

용어는 glossary(문구 사전) 기준으로 통일한다.

비목표(Non-goals)

Phase 1에서 "튜토리얼 10장" 같은 장황한 온보딩 금지

단축어/Controls 설정을 필수로 강제하는 UX 금지

잠금화면에서 목록 탐색/검색/정렬 UI 제공 금지

용어 정의

Onboarding: 첫 실행 후 잠금화면 가치가 보이기까지의 전체 흐름

Setup Completion: 위젯 추가 + 대표 프로젝트(pinned) 또는 Today 요약 기준 설정이 완료된 상태

Glanceable: 잠금화면에서 1~2초 내 이해되는 요약

결정 필요(Open decisions)

온보딩 화면을 앱 내부에서 "한 번만 보여줄지/항상 접근 가능한 가이드로 둘지"

앱 내부 "가이드/도움말"을 Settings에 둘지, 첫 화면에 둘지

영어(EN) 기본 제공 여부(이미 strings-localization에서 다룸)

변경 시 파급효과(필수 동반 수정)

온보딩/설정 흐름 변경 시:

strings-localization.md 문구 키/길이 예산 동시 수정

testing.md의 수동 QA(C 항목) 동시 수정

release-lite 제출 문서(심사관 메모) 반영 필요

────────────────────────────────────────
A. 핵심 사용자 흐름(3개) — LivePlan 표준
────────────────────────────────────────

A1. 첫 실행(2분 내 가치 노출) — 필수 흐름
목표: "잠금화면에서 요약이 보인다"를 사용자에게 경험시키기

표준 흐름(최소 단계)

앱 실행 → "LivePlan 핵심 1문장" + 프라이버시 안내(마스킹 기본)

프로젝트 생성(시작일 필수, 마감일 선택)

태스크 2개 추가(일반 1, 매일 반복 1)

"위젯 추가 안내" 화면(짧게) + "완료"

잠금화면에서 위젯 확인(성공)

필수 안내 문구(요지)

"잠금화면은 주변 사람이 볼 수 있습니다. 기본은 마스킹 표시입니다."

"위젯은 iOS 정책에 따라 즉시 갱신되지 않을 수 있습니다."

A2. 매일 사용(반복 체크 + 우선순위) — 핵심 습관
목표: 매일 반복 항목과 미완료 요약을 "잠금화면에서" 처리(또는 앱 열어 최소 조작)

표준 흐름

잠금화면 위젯 확인 → (iOS 18+) Controls로 CompleteNextTask 또는 앱 열어 체크

반복 항목: "오늘 완료" 처리 후 잠금화면에서 사라짐(다음날 리셋)

A3. 급한 캡처(QuickAdd) — 인박스/핀 프로젝트로 빠르게
목표: "해야 할 일이 떠오르면 즉시 저장"

Controls/단축어/앱 UI 어디서든 QuickAdd 가능

기본 정책은 product-decisions.md(핀 우선, 없으면 Inbox)

────────────────────────────────────────
B. 설정 완료율을 높이는 UX 규칙
────────────────────────────────────────

B1. 위젯 추가는 "필수 안내, 선택 강제 아님"

위젯은 LivePlan의 핵심 가치이므로 반드시 안내하지만, 강제로 추가하게 만들지 않는다.

안내는 "2문장 + 1개 행동 버튼(설정 방법 보기)"을 넘기지 않는다.

B2. Live Activity/단축어는 "선택 기능"으로 분리

단축어 8시간 갱신은 선택 기능이다(product-decisions.md).

앱이 없어도 위젯으로 핵심 기능이 유지되도록 설계한다(폴백).

B3. pinned 설정은 "추천"

대표 프로젝트(pinned)는 UX를 안정화하지만, pinned 없음 상태도 자연스럽게 todayOverview로 폴백한다.

pinned이 없으면 "추천 배너/설정 힌트" 정도만 제공(강제 금지).

────────────────────────────────────────
C. 잠금화면 문구 기준(요약 + 일관성)
────────────────────────────────────────

C1. 용어 통일

"미완료/지연/임박/반복/대표 프로젝트" 등 표준 용어만 사용(strings-localization.md의 glossary 준수).

C2. 길이 예산 준수

잠금화면 문구는 strings-localization.md의 길이 예산을 따른다.

Level 1/2에서는 제목 원문을 포함하지 않는다.

C3. 폴백/오류 문구 원칙

사용자 책임을 암시하지 않는다("당신이 설정하지 않아서…" 금지).

짧고 행동 유도형("앱을 열어 확인", "프로젝트를 추가")을 사용한다.

────────────────────────────────────────
D. 엣지 상태 UX(반드시 정의)
────────────────────────────────────────

D1. 데이터 없음(프로젝트/태스크 0)

위젯: "할 일을 추가하세요"

앱: "프로젝트 만들기" CTA 1개

D2. pinned 없음

위젯: todayOverview 폴백, "대표 프로젝트 설정하면 더 정확해집니다"는 앱 내부 힌트로만

D3. Live Activity 꺼짐/권한 제한

폴백: 위젯 요약으로 가치 유지

앱: "Live Activity는 선택 기능" 설명 + 켜는 방법 링크(선택)

D4. 단축어 자동화 미작동

폴백: 다음 실행 시 회복을 기대(멱등성)

사용자 안내: "필수 아님, 위젯만으로도 동작"

끝.
