목적

LivePlan을 App Store에 제출할 때 필요한 "설명/재현/표기"를 고정하여 심사 지연/리젝 리스크를 낮춘다.

특히 LivePlan의 특성(잠금화면/Live Activity/단축어 자동화)을 과장 없이, 심사관이 재현 가능하게 안내한다.

적용 범위

App Store Connect 메타데이터(앱 설명/키워드/스크린샷 캡션/릴리즈 노트)

App Review Notes(심사관 재현 단계)

권한/프라이버시 표기(App Privacy 포함)

TestFlight 릴리즈 노트

상위 규칙 우선순위

product-decisions.md(선택 기능/기본값 확정)

lockscreen.md/intents.md(실제 동작 기준)

release-lite 산출물(최종 문서)

본 문서(appstore-submission)

핵심 원칙(강행)

"선택 기능"을 명확히 표기

단축어 8시간 갱신은 선택 기능이다.

Live Activity가 없어도 위젯으로 핵심 기능이 유지된다는 폴백을 명시한다.

심사관 재현 단계는 최소 5단계로

심사관이 따라할 수 있도록, 앱 내부 버튼/경로를 명확히 제시한다.

프라이버시 기본값(마스킹 ON) 명시

잠금화면 노출 최소화를 기본으로 한다는 점을 설명한다.

과장/모호한 표현 금지

"항상 실시간" "무조건 즉시 업데이트" 같은 표현 금지(위젯 정책상 불가).

스크린샷 스토리보드(권장 5~7장)

프로젝트/할 일 생성(시작일/반복 포함)

잠금화면 위젯 요약(Top N + 카운트)

반복 항목(오늘 완료 → 내일 리셋) 컨셉

프라이버시 모드(마스킹)

Live Activity(요약 + 1개 핵심, 선택 기능)

단축어 자동화(8시간 갱신, 선택 기능)

iOS 18 Controls(완료/추가, 선택 기능)

App Review Notes(필수 포함 요소)

앱 목적 1문장

위젯/Live Activity/단축어의 관계(필수 vs 선택)

재현 단계(프로젝트 생성 → 태스크 추가 → 위젯 확인 → 완료 처리 → Live Activity/단축어 선택)
