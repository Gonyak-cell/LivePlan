# documentation.md

## 목적

LivePlan 프로젝트의 문서 작성 및 업데이트 규칙을 고정한다.

## 적용 범위

- `.claude/rules/*.md` (규칙 문서)
- `Docs/*.md` (설계/로드맵 문서)
- `README.md` 및 기타 프로젝트 문서

## 핵심 원칙(강행)

### 1. 업데이트 시간 기재 형식

문서 업데이트 시 날짜와 시간을 **초 단위까지** 기재한다.

**형식**: `YYYY-MM-DD HH:mm:ss` (24시간제, 로컬 타임존 기준)

**예시**:
- `Last Updated: 2026-02-02 14:35:42`
- `Updated: 2026-02-02 09:12:05`

**금지**:
- 날짜만 기재 (`2026-02-02`) ❌
- 시:분만 기재 (`2026-02-02 14:35`) ❌

### 2. 업데이트 기록 위치

- 문서 상단(제목 바로 아래) 또는 문서 하단에 일관되게 배치
- 동일 문서 내에서는 위치를 변경하지 않음

### 3. 변경 이력(선택)

중요한 변경의 경우 변경 내용을 간략히 기록할 수 있다.

**예시**:
```
## 변경 이력
- 2026-02-02 14:35:42: 업데이트 시간 기재 규칙 추가
- 2026-02-01 10:22:18: 초기 문서 생성
```

끝.
